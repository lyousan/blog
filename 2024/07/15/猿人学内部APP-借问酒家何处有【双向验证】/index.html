猿人学内部APP-借问酒家何处有【双向验证】

app版本：1.0.50

抓包工具：reqable（有坑）、charles、proxydroid

hook工具：frida

keystore-explorer：https://keystore-explorer.org/downloads.html

在线证书转换：https://myssl.com/cert_convert.html

keytool BKS支持：https://www.bouncycastle.org/download/bouncy-castle-java-lts/#download

0x0 先附上一张抓包成功的截图

0x1 绕过sslpinning

frida脚本下载地址：https://codeshare.frida.re/@akabe1/frida-multiple-unpinning/

0x2 导出客户端证书

继续站在大佬们的肩膀上 证书hook脚本下载地址：android-keystore-audit/frida-scripts/tracer-keystore.js at master · WithSecureLabs/android-keystore-audit · GitHub

以spwan模式执行后就能hook到证书的加载，包括证书的密码和证书的字节流

证书文件可以在assets下找到clientCA.bks，也可以修改这个脚本，在load之前将inputStream保存到文件中。

随后将导出的证书移动到sdcard目录后使用adb pull将证书拉取到电脑上。

0x3 证书转换

猿人学的这张证书是bks格式的，reqable和charles并不支持，需要转换成p12或者pem格式后才能使用。

  • 使用keystore-explorer进行转换，下载地址：KeyStore Explorer - Download

  • 使用jdk自带的keytool进行转换，建议jdk8

这里使用keystore-explorer的时候没有识别出猿人学的bks证书，然后采用了keytool的方式。

你有可能会遇到keytool 错误: java.security.KeyStoreException: BKS not found这个问题，需要单独下载一个jar包来支持BKS。

下载地址：https://www.bouncycastle.org/download/bouncy-castle-java-lts/#download

然后为keytool指定provider和providerpath参数

0x4 抓包软件配置

如果你使用reqable的话，这里不要使用协同抓包，会跟frida有冲突，可以单独配置代理，建议用vpn代理抓包，这里我采用的方案是proxydroid+charles。

  1. 导入客户端证书

    填好服务器主机和端口，然后导入我们上一步转换的p12格式的证书即可。

  2. 加入到ssl代理中

    这道题的接口并不是443端口，如果你遇到抓包的内容都是乱码，那么你还需要添加ssl代理的端口

0x5 代码请求

这里给出两种携带证书的请求方式

  1. requests    pip install requests

    这里需要将p12证书转换成pem，在线转换地址：https://myssl.com/cert_convert.html

    转换完之后会有一个压缩包，解压里面就有.cer和.key文件。

  2. requests_pkcs12    pip install requests_pkcs12
